# 首席审判专家 (Chief Presiding Justice) — Skill README

## 简介

**首席审判专家**是一个面向中国法律实务的智能审判辅助技能。它将 Claude 定位为合议庭承办法官，能够自动读取用户上传的案件材料，按照中国法院标准公文规范，生成逻辑严密、法律引用规范的**审理报告**。

---

## 适用场景

| 案件类型 | 典型用途 |
|---|---|
| 民事案件 | 合同纠纷、侵权责任、相邻权纠纷等 |
| 商事案件 | 买卖合同、公司股权、金融借贷等 |
| 知识产权案件 | 著作权、专利权、商标权侵权认定 |
| 行政案件 | 行政许可、行政处罚合法性审查 |
| 刑事案件 | 辅助分析证据充分性与构成要件 |

---

## 触发方式

### 方式一：关键词触发
在上传案件材料后，输入以下任意指令：

- `请生成审理报告`
- `分析案卷`
- `归纳争议焦点`
- `法律意见书`
- `证据分析`
- `案件梳理`

### 方式二：文件上传触发
上传以下任意类型的法律相关文件后，Claude 将主动询问是否需要生成审理报告：

- 📄 PDF / Word 文档（合同、协议、起诉书、判决书）
- 🗒️ 庭审笔录
- 🖼️ 图片类证据（照片、截图）
- 📊 账单、往来函件、鉴定意见

---

## 工作流程

```
上传材料
    │
    ▼
① 材料扫描与分类
   识别证据目录、庭审笔录、合同文本等，发现缺口时主动提示
    │
    ▼
② 事实萃取
   提取各方陈述、无争议事实、关键时间节点
    │
    ▼
③ 争议焦点归纳
   识别合同效力、侵权责任、数额计算等核心争点
    │
    ▼
④ 必要事实穿透检查
   对每项请求权核查证据支撑，发现证据缺口时暂停并提示
    │
    ▼
⑤ 法律检索（联网）
   核实条文现行有效状态，处理时际法律冲突
    │
    ▼
⑥ 生成标准格式审理报告
```

---

## 输出报告结构

生成的审理报告包含以下六大部分：

1. **诉讼请求** — 逐项列明原告（及反诉）请求
2. **庭审核实的事实** — 已查明事实 + 存疑事实（注明举证责任分配）
3. **证据质证与采信** — 表格化呈现各方证据及本院认定
4. **争议焦点** — 简洁归纳各焦点
5. **本院认为** — 逐一进行法律发现 → 事实对应 → 法理深化 → 指导性案例参照
6. **预判结论** — 对每项诉请给出支持/驳回预判，涉金额者列明计算公式

## 法律引用规范

本技能严格遵循**效力位阶**引用法律：

```
第一顺位：法律（全国人大及常委会）
  └─ 格式：《中华人民共和国民法典》第X条第X款之规定
第二顺位：行政法规（国务院）
第三顺位：司法解释（须注明文号，如"法释〔2023〕X号"）
第四顺位：指导性案例（须阐明与本案相似性逻辑）
补充参考：典型案例、学术观点（明确标注，严禁与生效法律混淆）
```

**强制规则**：
- 禁止简称，首次引用必须使用全称
- 引用具体到条、款、项、目
- 引用前联网确认条文现行有效，《民法典》施行后旧法已废止
- 学术观点须标注"法理学界主流观点认为……"

## 交互特性

| 特性 | 说明 |
|---|---|
| 语言 | 全程中文，符合中国法院公文规范 |
| 称谓 | 报告使用"本院"，不使用第一人称"我" |
| 证据缺口处理 | 发现必要事实缺失时，暂停生成并发出【事实缺失预警】，等待用户确认后继续 |
| 时际法律冲突 | 主动识别旧法/新法适用问题，并征询用户是否需要对比分析 |
| 不确定性标注 | 无法查明的事实明确注为"该事实存疑"，不擅自推断 |
| 合规免责 | 每份报告末尾附注辅助工具声明 |

## 使用示例

**用户操作**：
1. 上传一份买卖合同纠纷的庭审笔录（PDF）及证据目录
2. 输入：`请生成审理报告`

**Claude 响应流程**：
- 扫描材料，列出已识别文件清单
- 若发现卖方付款凭证缺失，输出【事实缺失预警】
- 用户确认"继续"后，联网检索《民法典》相关条文
- 输出完整格式审理报告，包含证据质证表、争议焦点、本院认为、预判结论

---

## 注意事项

> ⚠️ **本技能生成的审理报告为辅助审判分析工具，最终裁判以合议庭评议结果为准。**
> 
> 刑事案件分析仅供辅助参考，最终认定须经严格法定程序。
